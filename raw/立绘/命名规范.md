# 立绘文件命名规范

每个卡池角色的标准文件集如下。

## 主立绘 / 状态节点

| 节点 | 命名 | 文件数 | 说明 |
|---|---|---|---|
| **初始** | `{角色}_初始.png` | 1 | 主立绘，无 L2D 版本 |
| **洞悉** | `{角色}_洞悉_立绘.png` + `{角色}_洞悉_L2D.png` | 2 | 立绘版 + Live2D 版 |
| **尤提姆** | `{角色}_尤提姆.png` | 1 | 单文件，无 L2D 版本（部分角色无） |
| **签名** | `{角色}_签名.png` | 1 | 卡池角色的签名小图 |

## 单品（初始三件）

随主立绘附带的三件代表性物品：

```
初始_{单品名1}.png
初始_{单品名2}.png
初始_{单品名3}.png
```

固定 3 张。

## 时装（衣着）

每套时装（"衣着 N"）固定结构 = **2 张立绘 + 3 件单品**：

```
衣着_{衣着名}_立绘.png        ← 时装立绘版
衣着_{衣着名}_L2D.png         ← 时装 Live2D 版
{衣着名}_{单品1}.png          ← 对应单品 1
{衣着名}_{单品2}.png          ← 对应单品 2
{衣着名}_{单品3}.png          ← 对应单品 3
```

每多 1 套时装就多这 5 张。

## 完整张数公式

设角色有 N 套时装：

```
总张数 = 主立绘(初始 1 + 洞悉 2 + 尤提姆 1) + 签名(1) + 初始单品(3) + 时装(5×N)
       = 8 + 5N
```

- 0 套时装：8 张
- 1 套时装：13 张
- 2 套时装：18 张
- 3 套时装：23 张

（部分角色无尤提姆，扣 1 张；部分另有基础版等扩展）
