# 重返未来：1999 剧情 Wiki — Schema

> 本文件定义了 LLM 维护此 Wiki 的规则和约定。
> 所有 Wiki 操作（Ingest / Query / Lint）都应遵循此文档。

## 项目概述

这是《重返未来：1999》（Reverse: 1999）的剧情知识库，用于同人创作参考。
Wiki 基于游戏的官方剧情文本构建，涵盖主线故事、活动剧情、角色剧情、轶事，以及小径、黄昏的音序（Roguelike）、第三扇门等世界观设定与故事碎片。

## 目录结构

```
raw/                    # 原始素材（不可变）
  ├── 主线故事/          # 主线章节文本
  ├── 活动剧情/          # 活动事件文本
  ├── 角色剧情/          # 角色个人故事
  ├── 角色/             # 所有角色的页面资料
  ├── 小径/             # 小径的故事内容与补充设定
  ├── 黄昏的音序/         # 肉鸽剧情，主要是塞梅尔维斯为主角的故事
  ├── 第三扇门/           # 探索解密相关的世界观碎片
  ├── 轶事/             # 短篇故事
  └── story-links.json  # 所有链接的索引

wiki/                   # LLM 生成的知识库
  ├── index.md          # 全局索引（必须保持更新）
  ├── log.md            # 操作日志（仅追加）
  ├── 角色/             # 角色百科页面
  ├── 世界观/            # 世界设定（暴风雨、神秘学、时间倒流等）
  ├── 地点/             # 故事发生的地点
  ├── 组织/             # 基金会、重塑之手等组织
  ├── 主题/             # 主题分析
  └── 剧情概要/          # 剧情线索梳理
```

## 页面约定

### Frontmatter（YAML）

所有 Wiki 页面必须包含 YAML frontmatter：

```yaml
---
type: "character" | "location" | "organization" | "concept" | "summary" | "theme"
title: "页面标题"
aliases: ["别名1", "别名2"]   # 可选
tags: ["标签1", "标签2"]      # 可选
sources: ["序章-01", "1ST-03"] # 引用的原始素材
updated: "YYYY-MM-DD"
---
```

### 交叉引用

使用 Obsidian 兼容的 Wiki Link 语法：
- `[[角色/维尔汀]]` — 链接到角色页面
- `[[地点/圣帕弗利]]` — 链接到地点页面
- `[[角色/维尔汀|维尔汀]]` — 带显示文本的链接

## 工作流

### Ingest（摄入新素材）

1. 阅读 `raw/` 中的章节文本
2. 识别出现的角色、地点、组织、概念
3. 创建或更新对应的 Wiki 页面
4. 更新 `wiki/index.md`
5. 在 `wiki/log.md` 追加记录

每次 Ingest 应处理 5-10 个章节（一个完整小节或事件），确保上下文连贯。

### Query（查询）

1. 先读 `wiki/index.md` 定位相关页面
2. 读取相关 Wiki 页面
3. 如需原文，查阅 `raw/` 中的原始素材
4. 有价值的分析结果应保存为新的 Wiki 页面

### Lint（健康检查）

定期检查：
- 孤立页面（无入链）
- 缺失页面（被引用但不存在）
- 过时信息（新章节可能修正旧设定）
- 交叉引用完整性

## 命名规范

- 角色：使用游戏官方中文名（如"维尔汀"而非"Vertin"）
- 地点：使用游戏官方翻译
- 文件名：中文，无空格（用下划线替代）
- 所有 Wiki 内容使用中文撰写

## 同人创作重点

Wiki 的首要目的是支持同人创作，因此特别关注：
1. **角色说话风格** — 语气、用词习惯、口头禅
2. **角色关系** — 互动方式、情感连接
3. **世界观设定** — 暴风雨机制、神秘学体系、时代背景
4. **角色性格** — 在不同情境下的反应和决策模式
5. **时间线** — 事件发生的先后顺序和因果关系
